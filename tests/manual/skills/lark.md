# 技能：飞书 / Lark（com.electron.lark）· 把链接粘到与某人的聊天框（绝不发送）

Lark 是 Electron，actuator 已强开 `AXManualAccessibility`，AX 树丰富但有几个**必须知道的坑**：

- **文字输入只认 `ax_set_value`（直接写元素值）和 `cmd+v` 粘贴**；逐字 `type`（合成按键）在 Lark **不生效**（被丢弃）。
- **搜索框**：`ax_set_value` 只改值、**不触发**搜索的 onChange → 搜不出结果。若要用搜索，setvalue 后必须补一个**真实输入事件**才会刷新结果：要么 `ax_set_value` 清空再 `cmd+v` 粘贴人名，要么补一个字符键入。
- **当前会话身份**：消息输入框（`AXTextArea`，宽>100）为空时 AXValue 不带对方名字，**不能靠输入框判断在跟谁聊**。要判断当前会话，看消息区/会话标题的文本。

## 把链接粘到与「<某人>」的聊天框
1. **打开这个人的会话** —— 关键：界面里同一个人名会出现在多处（会话列表行、聊天标题、聊天记录），
   **必须用 `ax_find("lark", "<人名>")`**（不要用 ocr_locate——它分不清）。ax_find 会按**容器/区域**列出所有匹配，例如：
   ```
   [40] AXStaticText "温子墨" 容器A/会话列表 可见 bbox=[404,174,42,16]   ← 1对1会话行（要的就是它）
   [41] AXStaticText "温子墨: [图片]…" 容器A/会话列表 隐藏 …            （消息预览，不是入口）
   [42] AXStaticText "温子墨" 容器B/聊天区 …                          （聊天标题/记录，不是入口）
   ```
   选 **容器=会话列表、label 正好是人名本身（不是「人名: …」预览）、可见** 的那一行 → `clickxy` 它的 bbox 中心打开会话。
2. **确认切到了**：再 `ax_find("lark","<人名>")` 或 ax_snapshot，确认聊天区标题是这个人。
3. **粘链接**：找消息输入框 `AXTextArea`（宽>100）→ `ax_set_value index=N text="<链接>"` 写进草稿框。
   **绝不按回车、绝不点发送** —— 只放进输入框，等用户自己发。

## 安全
- 只允许粘到被授权的那个人的会话；切错人/没确认到这个人，就**别粘**，把链接留在剪贴板交给用户。
- 全程不发送任何消息、不点删除。
